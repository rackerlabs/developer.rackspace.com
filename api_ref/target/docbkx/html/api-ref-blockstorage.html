Cloud Block Storage API v1

Volumes

POST
v1/​{tenant_id}​/volumes

Creates a volume.

 

The examples in the following table create a 100 GB SATA volume called vol-001.

Normal Response Codes
200
Request parameters
Parameter Style Type Description
volume plain string

A partial representation of a volume used in the creation process. This parameter is required for this operation.

size plain string

The size (in GB) of the volume. This is a required parameter. Note: When creating a volume from a snapshot, the volume size must be greater than the snapshot size.

display_description (Optional) plain string

A description of the volume. This parameter is optional.

display_name (Optional) plain string

The name of the volume. This parameter is optional.

snapshot_id (Optional) plain string

The optional snapshot from which to create a volume. This parameter is optional.

volume_type (Optional) plain string

The type of volume to create, either SATA or SSD. This parameter is optional. If not defined, then the default, SATA, is used.

source_volid (Optional) plain string

The source identifier of an existing Cloud Block Storage volume that you want to clone (copy) to create a new volume. This parameter is optional.

availability_zone (Optional) plain string

This parameter is no longer used. Therefore, you can supply any value for availability_zone. If you specify no value, it defaults to nova.

metadata (Optional) plain string

This optional parameter is available if you want to set any metadata values on the volume.

Snapshots

POST
v1/​{tenant_id}​/snapshots

Creates a snapshot.

 

A snapshot is a point-in-time copy of the volume. You must flush all writes to the volume before you create a snapshot. To do so, either unmount any file systems on the volume or detach the volume.

Snapshots are incremental, so each time that you create a snapshot, the incremental changes for the new snapshot are appended to the previous snapshot, which is still available. Note that you can create a volume from the snapshot if desired.

The os-extended-snapshot-attributes:progress field in the response body shows the snapshot progress.

When the POST operation returns the 201 response code, the snapshot is complete.

Normal Response Codes
201
Request parameters
Parameter Style Type Description
snapshot plain string

A partial representation of a snapshot used in the creation process.

volume_id plain string

The ID of the volume to snapshot.

force (Optional) plain string

[True/False] Indicate whether to snapshot, even if the volume is attached. Default==False.

display_name (Optional) plain string

Name of the snapshot. Default==None.

display_description (Optional) plain string

Description of snapshot. Default==None. 

{
    "snapshot": {
        "display_name": "snap-001",
        "display_description": "Daily backup",
        "volume_id": "521752a6-acf6-4b2d-bc7a-119f9148cd8c",
        "force": true
     }
}
